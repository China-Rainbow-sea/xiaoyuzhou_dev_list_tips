# 项目说明文档

## 1. 技术选型



- **编程语言**：JavaScript (原生)，理由：无需编译，直接在浏览器中运行，实现简单，开发效率高。
- **框架/库**：原生 JavaScript (不依赖框架)，理由：项目规模吉小，原生JS完全足够实现，避免了引入框架的额外开销，学习成本低。  
- **数据库/存储**：浏览器 localStorage，理由：轻量、简单，适合本地单用户应用，无需额外的服务器或数据库配置。
- **替代方案对比**：  
  - 与使用 React/Vue 等框架相比：原生JS实现更加轻量，无需构建工具和依赖安装，打开HTML文件即可使用。  
  - 与使用 IndexedDB 相比：localStorage 更简单易用，对于当前任务数据量完全足够，虽然 IndexedDB 支持更大数据量和更复杂查询，但在这个场景下是过度设计了。
  - 与使用后端数据库相比：纯前端实现无需部署服务器，快速实现。

## 2. 项目结构设计

- 整体架构说明（纯前端单页应用，采用经典的 HTML + CSS + JavaScript 三层结构。HTML 提供页面结构，CSS 负责样式呈现，JavaScript 处理业务逻辑和数据交互。

- 目录结构示例：

  ```
  src/
    index.html
    style.css
    app.js
  ```

  

- **模块职责说明**：  

- **index.html**：定义应用的 DOM 结构，包括任务表单等一些界面元素。  

- **style.css**：提供响应式设计和良好的用户界面。  

- **app.js**：包含所有业务逻辑，如任务的增删改查、数据持久化、筛选排序等功能的实现。

## 3. 需求细节与决策

**描述是否必填？如何处理空输入？**：任务描述为可选字段，只有任务标题是必填项。

当用户尝试添加没有标题的任务时，系统会弹出提示框要求用户输入标题，并将焦点设置回标题输入框。

**已完成的任务在 UI 中如何显示？**：已完成的任务在 UI 上会通过以下方式区分：

- 任务标题会添加删除线样式
- 任务内容颜色变浅
- 任务旁边的复选框会显示为选中状态

**任务排序逻辑**：

- 默认按照创建时间降序排列（最新创建的任务在最上方）
- 用户可以切换为按创建时间升序排列
- 任务排序会与筛选和搜索功能结合使用



**扩展功能设计思路**：

- **搜索功能**：实现了基于标题和描述的全文搜索，帮助用户快速定位任务。
- **数据持久化**：使用 localStorage 保存任务数据，确保页面刷新或关闭后数据不会丢失。

## 4. AI 使用说明



**是否使用 AI 工具？**：是，使用了 AI 辅助编程工具。  

**使用 AI 的环节**：  

- 代码片段生成：生成了基础的 HTML 结构、CSS 样式和 JavaScript 函数。  

- 功能实现：辅助实现了任务过滤、排序和本地存储的核心逻辑。    

**AI 输出如何修改**：

- 对 AI 生成的 UI 样式进行了调整，优化了响应式设计和视觉体验。
- 增强了数据处理逻辑，添加了输入验证和错误处理机制。
- 改进了用户交互体验，增加了空状态提示和确认对话框。

## 5. 运行与测试方式

- **本地运行方式**：无需安装依赖，直接用浏览器打开 index.html 文件即可运行。
- Chrome 、Firefox 、Edge  等主流浏览器
- Windows操作系统

**已知问题与不足**：

* localStorage 有大小限制（约 5MB），如果任务数量极多可能会超出限制。

- **如果有更多时间，你会如何改进？**：

* 添加任务编辑功能，允许用户修改已有任务的标题和描述。
